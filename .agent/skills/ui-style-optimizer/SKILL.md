# UI 样式优化

## 目标

- 在不改变业务逻辑的前提下，提升视觉层级、可读性与一致性
- 优先做“全局变量/组件基类”层面的改进，减少到处改 class 的维护成本

## 适用场景（触发条件）

- 用户要求“继续优化 UI / 调整样式 / 视觉更高级”
- 用户提供截图并指出具体问题（白边明显、对比度低、间距拥挤、按钮太大/太小）
- 同一页面多个组件风格不一致，需要统一玻璃变量、文本变量、阴影/圆角体系

## 输入要求（尽量一次收集齐）

- 截图或明确位置（哪个模块/组件/页面）
- 期望风格关键词（更通透/更暗/更拟物/更扁平/更紧凑/更松弛）
- 目标设备（桌面/手机/电视）与分辨率范围

## 工作流程

1. 定位渲染源
   - 找到相关组件（如 AppGrid/SettingsModal/Widget）与 CSS 入口（如 src/style/index.css）
2. 建立“设计令牌”
   - 优先补齐/调整语义变量（--text-primary/secondary/muted、--glass-bg-*、--theme-*）
3. 组件级优化
   - 调整卡片：padding、min-height、圆角、边框透明度、hover/active 层级
   - 调整标题：字重、字距、图标底座、对齐与间距
4. 验证
   - 启动 dev 预览，检查桌面/移动端断点
   - 运行 build，确保无类型/构建错误

## 输出格式（交付物）

- 简要说明：改了哪些视觉点、为什么这样改
- 代码引用：给出修改文件与关键行范围
- 预览方式：本地预览地址或复现步骤

## 约束

- 不新增注释（除非用户明确要求）
- 不引入新依赖（除非仓库已使用并且用户同意）
- 优先小步可回退：集中修改在少数文件（index.css + 目标组件）

## 示例

用户：白边太明显了，卡片看起来廉价  
执行：
- 下调边框/高亮透明度，增加 hover 阴影层次
- 补齐 --glass-bg-hover / --glass-border 与 --text-* 语义变量

用户：应用卡片太扁，间距挤  
执行：
- 提升 card padding、min-height、圆角一致性
- 调整 grid gap（移动端更紧凑、桌面更舒展）
